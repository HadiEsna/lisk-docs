Style guide for writing good documentation content

The style guide provides best practices and guidelines, how to write helpful content in the Lisk documentation.

Best practices

Grammar & punctuation

The Lisk documentation is following the Tips on Grammar, Punctuation and Style from the Harvard College Writing Center.

Tone of voice

The following guidelines should be followed:

  • Style and tone

  • Accessible content

  • Writing for a global audience

  • Writing inclusive documentation

Scannability

The Lisk documentation is also following the scannable content guide from the Microsoft documentation.

Links

The following guidelines should be followed:

  • Cross-references

  • Link texts
